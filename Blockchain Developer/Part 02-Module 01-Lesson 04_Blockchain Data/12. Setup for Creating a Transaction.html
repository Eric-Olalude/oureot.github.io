12. Setup for Creating a Transaction

Objectives

  • The objective is to create a raw transaction.
  • Bitcoin transaction inputs come from the outputs of a previous transaction.
  • During a transaction, these funds are transferred from one address to another address.
  • When it comes time to spend a transaction, our wallet scans the blockchain for any spendable UTXO that are tied to our wallet address.
  • For this exercise, to create a raw transaction, we will use the outputs from that transaction as our starting point inputs.

C2L4 SetUp V2

Environment Setup:

  • Start your Bitcoin Core Software in the testnet mode

  • Note: In a previous lesson here, we used our wallet to create a transaction from the GUI.

    • If your wallet doesn’t already have available transactions with outputs to spend, checkout the link above to create a transaction so you have outputs to work with.

  • Make sure you have a receiving address, that you are going to send BTCs to. This can be an external address, or you can have more than 1 receiving addresses in your wallet.

  • To create a new receiving address:

    • Open the Debug Window in the Menu -> Help -> Debug Window.
    • In the Debug Window open the Console tab.
    • Type in the command line getnewaddress [label]. You can also add a label for your address by typing, getnewaddress test.
    • This automatically creates a new receiving address in your wallet.
